# Gemini Project Guidelines

이 파일은 Gemini CLI 에이전트가 이 프로젝트를 유지보수할 때 반드시 지켜야 할 가이드라인입니다.

## 1. UI/UX 디자인 원칙
- **테마 일관성**: 항상 오버워치 2의 시각적 정체성을 유지하십시오. 배경색은 `bg-slate-950`, 강조 색상은 오렌지(`text-orange-500`, `#f99e1a`)를 사용합니다.
- **반응형 보장**: 다양한 화면 크기에서도 레이아웃이 무너지지 않아야 하며, 특히 가로 스크롤은 절대 발생하지 않도록 관리하십시오.
- **글래스모피즘**: 패널과 버튼에는 투명도와 블러 효과를 적절히 활용하여 깊이감을 부여하십시오.

## 2. 기술 및 코드 컨벤션
- **Tailwind CSS**: 스타일링은 반드시 Tailwind CSS의 유틸리티 클래스만을 사용하여 작성하십시오. 인라인 스타일이나 외부 CSS 파일 생성을 지양합니다.
- **TypeScript**: 모든 데이터 구조와 컴포넌트 프롭스에는 엄격한 타입을 적용하여 안정성을 확보하십시오.

## 3. 데이터 및 비즈니스 로직
- **데이터 소스**: 영웅의 목록, 이름, 초상화, 역할군 등의 메타데이터는 OverFast API(`https://overfast-api.tekrop.fr/heroes`)를 우선적으로 사용하십시오.
- **카운터 로직**: 영웅 간의 상성 매핑 데이터는 `src/constants.ts`의 `counterMap`을 최우선으로 따르며, 새로운 영웅이 추가될 경우 이 파일을 업데이트하십시오.
